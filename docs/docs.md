# Accessibility im Web und seine Standards

Barrierefreiheit im Web, auch bekannt als Web Accessibility, ist ein entscheidender Aspekt der modernen Webentwicklung. Sie stellt sicher, dass Menschen mit Behinderungen das Internet uneingeschränkt nutzen können. Dies umfasst Menschen mit Seh-, Hör-, motorischen oder kognitiven Beeinträchtigungen. Die Bedeutung der Barrierefreiheit im Web kann nicht genug betont werden, da das Internet eine zentrale Rolle in unserem täglichen Leben spielt, sei es für Bildung, Arbeit, soziale Interaktionen oder den Zugang zu Informationen.

Die Web Content Accessibility Guidelines (WCAG) sind die international anerkannten Standards für die Barrierefreiheit im Web. Diese Richtlinien wurden vom World Wide Web Consortium (W3C) entwickelt und bieten umfassende Anleitungen zur Verbesserung der Zugänglichkeit von Webinhalten. Die WCAG besteht aus vier Prinzipien: Wahrnehmbarkeit, Bedienbarkeit, Verständlichkeit und Robustheit. Diese Prinzipien stellen sicher, dass Webinhalte für alle Benutzer zugänglich und nutzbar sind.

# Themenübersicht

**Notwendige Themen für die Accessibility im Web**

## Wahrnehmbarkeit

- Textalternativen für Nicht-Text-Inhalte
- Untertitel und Transkripte für Multimedia-Inhalte
- Anpassbare Schriftgrößen und Kontraste

[Mehr erfahren](./docs-perceivable.md)

## Bedienbarkeit

- Tastaturzugänglichkeit für alle Funktionen
- Zeitgesteuerte Inhalte und deren Steuerung
- Vermeidung von Inhalten, die Anfälle auslösen könnten

[Mehr erfahren](./docs-operable.md)

## Verständlichkeit

- Klare und einfache Sprache
- Konsistente Navigation und Layouts
- Hilfestellungen und Anleitungen für komplexe Interaktionen

[Mehr erfahren](./docs-understandable.md)

## Robustheit

- Kompatibilität mit verschiedenen Browsern und assistiven Technologien
- Verwendung von standardkonformem HTML und ARIA-Rollen
- Regelmäßige Tests und Validierungen der Barrierefreiheit

[Mehr erfahren](./docs-robust.md)
