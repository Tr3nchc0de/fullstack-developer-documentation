# Accessibility im Web und seine Standards

Barrierefreiheit im Web, auch bekannt als Web Accessibility, ist ein entscheidender Aspekt der modernen Webentwicklung. Sie stellt sicher, dass Menschen mit Behinderungen das Internet uneingeschränkt nutzen können. Dies umfasst Menschen mit Seh-, Hör-, motorischen oder kognitiven Beeinträchtigungen. Die Bedeutung der Barrierefreiheit im Web kann nicht genug betont werden, da das Internet eine zentrale Rolle in unserem täglichen Leben spielt, sei es für Bildung, Arbeit, soziale Interaktionen oder den Zugang zu Informationen.

Die Web Content Accessibility Guidelines (WCAG) sind die international anerkannten Standards für die Barrierefreiheit im Web. Diese Richtlinien wurden vom World Wide Web Consortium (W3C) entwickelt und bieten umfassende Anleitungen zur Verbesserung der Zugänglichkeit von Webinhalten. Die WCAG besteht aus vier Prinzipien: Wahrnehmbarkeit, Bedienbarkeit, Verständlichkeit und Robustheit. Diese Prinzipien stellen sicher, dass Webinhalte für alle Benutzer zugänglich und nutzbar sind.

# Accessibility im Web

## Notwendige Themen für die Accessibility im Web

**Wahrnehmbarkeit**

- Textalternativen für Nicht-Text-Inhalte
- Untertitel und Transkripte für Multimedia-Inhalte
- Anpassbare Schriftgrößen und Kontraste

[Mehr über Barrierefreiheit im Web](./text-alternatives.md)

**Bedienbarkeit**

- Tastaturzugänglichkeit für alle Funktionen
- Zeitgesteuerte Inhalte und deren Steuerung
- Vermeidung von Inhalten, die Anfälle auslösen könnten

**Verständlichkeit**

- Klare und einfache Sprache
- Konsistente Navigation und Layouts
- Hilfestellungen und Anleitungen für komplexe Interaktionen

**Robustheit**

- Kompatibilität mit verschiedenen Browsern und assistiven Technologien
- Verwendung von standardkonformem HTML und ARIA-Rollen
- Regelmäßige Tests und Validierungen der Barrierefreiheit

**Rechtliche Anforderungen**

- Kenntnis der lokalen und internationalen Gesetze und Vorschriften zur Barrierefreiheit
- Einhaltung der Web Content Accessibility Guidelines (WCAG)
- Dokumentation und Nachweis der Barrierefreiheitsmaßnahmen

**Benutzerfeedback und Tests**

- Einbeziehung von Nutzern mit Behinderungen in den Entwicklungsprozess
- Durchführung von Usability-Tests mit assistiven Technologien
- Kontinuierliche Verbesserung basierend auf Benutzerfeedback

**Schulung und Bewusstsein**

- Schulung des Entwicklungsteams in Barrierefreiheit
- Sensibilisierung für die Bedeutung der Barrierefreiheit
- Förderung einer inklusiven Kultur innerhalb des Teams und der Organisation

Die Einhaltung dieser Standards ist nicht nur eine ethische Verpflichtung, sondern kann auch rechtliche Anforderungen erfüllen und die Benutzererfahrung für alle verbessern. Durch die Implementierung von Barrierefreiheit im Web können Entwickler sicherstellen, dass ihre Websites für ein breiteres Publikum zugänglich sind und eine inklusive digitale Welt fördern.

## Notwendige Themen für die Accessibility im Web

### Wahrnehmbarkeit

#### Textalternativen

##### Nicht-Text-Inhalte

Stellen Sie Textalternativen für alle Nicht-Text-Inhalte bereit, damit diese von assistiven Technologien interpretiert werden können.

#### Zeitbasierte Medien

##### Nur Audio und Nur Video (Vorgefertigt)

Stellen Sie eine Alternative für vorgefertigte Audio- und Videoinhalte bereit.

##### Untertitel (Vorgefertigt)

Stellen Sie Untertitel für vorgefertigte Audio- und Videoinhalte bereit.

##### Audiobeschreibung oder Medienalternative (Vorgefertigt)

Stellen Sie eine Audiobeschreibung oder eine alternative Medienform für vorgefertigte Inhalte bereit.

##### Untertitel (Live)

Stellen Sie Untertitel für Live-Audio- und Videoinhalte bereit.

##### Audiobeschreibung (Vorgefertigt)

Stellen Sie eine Audiobeschreibung für vorgefertigte Videoinhalte bereit.

##### Gebärdensprache (Vorgefertigt)

Stellen Sie eine Gebärdensprachübersetzung für vorgefertigte Inhalte bereit.

##### Erweiterte Audiobeschreibung (Vorgefertigt)

Stellen Sie eine erweiterte Audiobeschreibung für vorgefertigte Videoinhalte bereit.

##### Medienalternative (Vorgefertigt)

Stellen Sie eine alternative Medienform für vorgefertigte Inhalte bereit.

##### Nur Audio (Live)

Stellen Sie eine Alternative für Live-Audioinhalte bereit.

#### Anpassbar

##### Informationen und Beziehungen

Stellen Sie sicher, dass Informationen und Beziehungen programmatisch bestimmt werden können.

##### Bedeutungsvolle Reihenfolge

Stellen Sie sicher, dass der Inhalt in einer sinnvollen Reihenfolge präsentiert wird.

##### Sensorische Merkmale

Stellen Sie sicher, dass Anweisungen nicht ausschließlich auf sensorischen Merkmalen basieren.

##### Orientierung

Stellen Sie sicher, dass die Orientierung der Inhalte nicht eingeschränkt ist.

##### Eingabezweck identifizieren

Stellen Sie sicher, dass der Zweck der Benutzereingabe programmatisch bestimmt werden kann.

##### Zweck identifizieren

Stellen Sie sicher, dass der Zweck von UI-Komponenten programmatisch bestimmt werden kann.

#### Unterscheidbar

##### Verwendung von Farbe

Stellen Sie sicher, dass Farbe nicht das einzige Mittel zur Übermittlung von Informationen ist.

##### Audiosteuerung

Stellen Sie sicher, dass Benutzer die Audiowiedergabe steuern können.

##### Kontrast (Minimum)

Stellen Sie sicher, dass der Text einen ausreichenden Kontrast zum Hintergrund hat.

##### Text vergrößern

Stellen Sie sicher, dass Text ohne Verlust von Inhalten oder Funktionalität vergrößert werden kann.

##### Bilder von Text

Vermeiden Sie die Verwendung von Bildern von Text, außer wenn ein bestimmter Stil erforderlich ist.

##### Kontrast (Erweitert)

Stellen Sie sicher, dass der Text einen erhöhten Kontrast zum Hintergrund hat.

##### Wenig oder kein Hintergrundaudio

Stellen Sie sicher, dass Hintergrundaudio leise ist oder ausgeschaltet werden kann.

##### Visuelle Präsentation

Stellen Sie sicher, dass Benutzer die visuelle Präsentation von Text anpassen können.

##### Bilder von Text (Keine Ausnahme)

Vermeiden Sie die Verwendung von Bildern von Text ohne Ausnahme.

##### Umbruch

Stellen Sie sicher, dass Inhalte ohne horizontales Scrollen angezeigt werden können.

##### Nicht-Text-Kontrast

Stellen Sie sicher, dass UI-Komponenten und grafische Objekte einen ausreichenden Kontrast haben.

##### Textabstand

Stellen Sie sicher, dass Benutzer den Textabstand anpassen können.

##### Inhalt bei Hover oder Fokus

Stellen Sie sicher, dass zusätzlicher Inhalt, der bei Hover oder Fokus angezeigt wird, zugänglich ist.

### Bedienbarkeit

#### Tastaturzugänglichkeit

##### Tastatur

Stellen Sie sicher, dass alle Funktionen über die Tastatur zugänglich sind.

##### Keine Tastaturfalle

Stellen Sie sicher, dass Benutzer nicht in einer Tastaturfalle gefangen sind.

##### Tastatur (Keine Ausnahme)

Stellen Sie sicher, dass alle Funktionen ohne Ausnahme über die Tastatur zugänglich sind.

##### Zeichenkurzbefehle

Stellen Sie sicher, dass Zeichenkurzbefehle deaktiviert oder angepasst werden können.

#### Ausreichend Zeit

##### Zeit anpassbar

Stellen Sie sicher, dass Benutzer die Zeitbegrenzungen anpassen können.

##### Anhalten, Stoppen, Ausblenden

Stellen Sie sicher, dass Benutzer bewegte, blinkende oder scrollende Inhalte anhalten, stoppen oder ausblenden können.

##### Keine Zeitbegrenzung

Stellen Sie sicher, dass Inhalte keine Zeitbegrenzung haben.

##### Unterbrechungen

Stellen Sie sicher, dass Benutzer Unterbrechungen steuern können.

##### Wiederherstellen der Authentifizierung

Stellen Sie sicher, dass Benutzer nach einer Authentifizierungssitzung ihre Daten wiederherstellen können.

##### Zeitüberschreitungen

Stellen Sie sicher, dass Benutzer über bevorstehende Zeitüberschreitungen informiert werden.

#### Anfälle und körperliche Reaktionen

##### Drei Blitze oder darunter

Stellen Sie sicher, dass Inhalte nicht mehr als drei Blitze pro Sekunde enthalten.

##### Drei Blitze

Stellen Sie sicher, dass Inhalte nicht mehr als drei Blitze pro Sekunde enthalten.

##### Animationen aus Interaktionen

Stellen Sie sicher, dass Benutzer Animationen aus Interaktionen deaktivieren können.

#### Navigierbar

##### Blöcke überspringen

Stellen Sie sicher, dass Benutzer Blöcke von Inhalten überspringen können.

##### Seitentitel

Stellen Sie sicher, dass jede Seite einen beschreibenden Titel hat.

##### Fokusreihenfolge

Stellen Sie sicher, dass die Fokusreihenfolge logisch und intuitiv ist.

##### Zweck des Links (Im Kontext)

Stellen Sie sicher, dass der Zweck jedes Links im Kontext bestimmt werden kann.

##### Mehrere Wege

Stellen Sie sicher, dass Benutzer mehrere Wege haben, um Inhalte zu finden.

##### Überschriften und Beschriftungen

Stellen Sie sicher, dass Überschriften und Beschriftungen beschreibend sind.

##### Fokus sichtbar

Stellen Sie sicher, dass der Tastaturfokus sichtbar ist.

##### Standort

Stellen Sie sicher, dass Benutzer ihren Standort innerhalb einer Website bestimmen können.

##### Zweck des Links (Nur Link)

Stellen Sie sicher, dass der Zweck jedes Links nur durch den Linktext bestimmt werden kann.

##### Abschnittsüberschriften

Stellen Sie sicher, dass Inhalte durch Abschnittsüberschriften strukturiert sind.

##### Fokus nicht verdeckt (Minimum)

Stellen Sie sicher, dass der Tastaturfokus nicht verdeckt ist.

##### Fokus nicht verdeckt (Erweitert)

Stellen Sie sicher, dass der Tastaturfokus nicht verdeckt ist.

##### Fokusdarstellung

Stellen Sie sicher, dass die Darstellung des Tastaturfokus deutlich sichtbar ist.

#### Eingabemodalitäten

##### Zeigergesten

Stellen Sie sicher, dass komplexe Zeigergesten nicht erforderlich sind.

##### Zeigerabbruch

Stellen Sie sicher, dass Benutzer Zeigeraktionen abbrechen können.

##### Beschriftung im Namen

Stellen Sie sicher, dass die Beschriftung von UI-Komponenten im Namen enthalten ist.

##### Bewegungsaktivierung

Stellen Sie sicher, dass Benutzer Bewegungsaktivierungen deaktivieren können.

##### Zielgröße (Erweitert)

Stellen Sie sicher, dass interaktive Ziele eine ausreichende Größe haben.

##### Gleichzeitige Eingabemechanismen

Stellen Sie sicher, dass Benutzer mehrere Eingabemechanismen gleichzeitig verwenden können.

##### Ziehbewegungen

Stellen Sie sicher, dass Benutzer Ziehbewegungen deaktivieren können.

##### Zielgröße (Minimum)

Stellen Sie sicher, dass interaktive Ziele eine Mindestgröße haben.

### Verständlichkeit

#### Lesbar

##### Sprache der Seite

Stellen Sie sicher, dass die Sprache der Seite programmatisch bestimmt werden kann.

##### Sprache der Teile

Stellen Sie sicher, dass die Sprache von Textteilen programmatisch bestimmt werden kann.

##### Ungewöhnliche Wörter

Stellen Sie sicher, dass ungewöhnliche Wörter erklärt werden.

##### Abkürzungen

Stellen Sie sicher, dass Abkürzungen erklärt werden.

##### Lesestufe

Stellen Sie sicher, dass Inhalte auf einer angemessenen Lesestufe präsentiert werden.

##### Aussprache

Stellen Sie sicher, dass die Aussprache von Wörtern erklärt wird.

#### Vorhersehbar

##### Bei Fokus

Stellen Sie sicher, dass keine unerwarteten Änderungen bei Fokus auftreten.

##### Bei Eingabe

Stellen Sie sicher, dass keine unerwarteten Änderungen bei Eingabe auftreten.

##### Konsistente Navigation

Stellen Sie sicher, dass die Navigation auf allen Seiten konsistent ist.

##### Konsistente Identifizierung

Stellen Sie sicher, dass Komponenten auf allen Seiten konsistent identifiziert werden.

##### Änderung auf Anfrage

Stellen Sie sicher, dass Änderungen nur auf Benutzeranfrage erfolgen.

##### Konsistente Hilfe

Stellen Sie sicher, dass Hilfe auf allen Seiten konsistent verfügbar ist.

#### Eingabeunterstützung

##### Fehlererkennung

Stellen Sie sicher, dass Fehler bei der Eingabe erkannt und dem Benutzer mitgeteilt werden.

##### Beschriftungen oder Anweisungen

Stellen Sie sicher, dass Beschriftungen oder Anweisungen für Eingabefelder bereitgestellt werden.

##### Fehlerhinweise

Stellen Sie sicher, dass Benutzer Vorschläge zur Fehlerbehebung erhalten.

##### Fehlervermeidung (Rechtlich, Finanziell, Daten)

Stellen Sie sicher, dass Benutzer Fehler bei rechtlichen, finanziellen oder datenbezogenen Transaktionen vermeiden können.

##### Hilfe

Stellen Sie sicher, dass Benutzer Hilfe bei der Eingabe erhalten können.

##### Fehlervermeidung (Alle)

Stellen Sie sicher, dass Benutzer Fehler bei allen Eingaben vermeiden können.

##### Redundante Eingabe

Stellen Sie sicher, dass Benutzer redundante Eingaben vermeiden können.

##### Zugängliche Authentifizierung (Minimum)

Stellen Sie sicher, dass Benutzer eine zugängliche Authentifizierungsmethode verwenden können.

##### Zugängliche Authentifizierung (Erweitert)

Stellen Sie sicher, dass Benutzer eine erweiterte zugängliche Authentifizierungsmethode verwenden können.

#### Kompatibel

##### Name, Rolle, Wert

Stellen Sie sicher, dass UI-Komponenten ihren Namen, ihre Rolle und ihren Wert programmatisch bestimmen lassen.

##### Statusmeldungen

Stellen Sie sicher, dass Statusmeldungen programmatisch bestimmt werden können.
